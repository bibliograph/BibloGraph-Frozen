Boolean

schema:DataType > schema:Boolean

Boolean: True or False.


Instances of schema:Boolean may appear as values for the following properties
PropertyOn TypesDescription
schema:acceptsReservations schema:FoodEstablishment Indicates whether a FoodEstablishment accepts reservations. Values can be Boolean, an URL at which reservations can be made or (for backwards compatibility) the strings Yes or No.
schema:free schema:PublicationEvent A flag to signal that the publication is accessible for free.
schema:isAvailableGenerically schema:Drug True if the drug is available in a generic form (regardless of name).
schema:isFamilyFriendly schema:CreativeWork Indicates whether this content is family friendly.
schema:isGift schema:Order Was the offer accepted as a gift for someone other than the buyer.
schema:isProprietary schema:Drug  or
schema:DietarySupplement 		True if this item's name is a proprietary/brand name (vs. generic name).
schema:multipleValues schema:PropertyValueSpecification Whether multiple values are allowed for the property. Default is false.
schema:readonlyValue schema:PropertyValueSpecification Whether or not a property is mutable. Default is false. Specifying this for a property that also has a value makes it act similar to a "hidden" input in an HTML form.
schema:representativeOfPage schema:ImageObject Indicates whether this image is representative of the content of the page.
schema:requiresSubscription schema:MediaObject Indicates if use of the media require a subscription (either paid or free). Allowed values are true or false (note that an earlier version had 'yes', 'no').
schema:valueAddedTaxIncluded schema:PriceSpecification Specifies whether the applicable value-added tax (VAT) is included in the price specification or not.
schema:valueRequired schema:PropertyValueSpecification Whether the property must be filled in to complete the action. Default is false.
