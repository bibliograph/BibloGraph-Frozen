URL

schema:DataType > schema:Text > schema:URL

Data type: URL.


Instances of schema:URL may appear as values for the following properties
PropertyOn TypesDescription
schema:acceptsReservations schema:FoodEstablishment Indicates whether a FoodEstablishment accepts reservations. Values can be Boolean, an URL at which reservations can be made or (for backwards compatibility) the strings Yes or No.
schema:additionalType schema:Thing An additional type for the item, typically used for adding more specific types from external vocabularies in microdata syntax. This is a relationship between something and a class that the thing is in. In RDFa syntax, it is better to use the native RDFa syntax - the 'typeof' attribute - for multiple types. Schema.org tools may have only weaker understanding of extra types, in particular those defined externally.
schema:applicationCategory schema:SoftwareApplication Type of software application, e.g. "Game, Multimedia".
schema:applicationSubCategory schema:SoftwareApplication Subcategory of the application, e.g. "Arcade Game".
schema:artform schema:VisualArtwork e.g. Painting, Drawing, Sculpture, Print, Photograph, Assemblage, Collage, etc.
schema:codeRepository schema:Code Link to the repository where the un-compiled, human readable code and related code is located (SVN, github, CodePlex).
schema:contentUrl schema:MediaObject Actual bytes of the media object, for example the image file or video file.
schema:discussionUrl schema:CreativeWork A link to the page containing the comments of the CreativeWork.
schema:downloadUrl schema:SoftwareApplication If the file can be downloaded, URL to download the binary.
schema:embedUrl schema:MediaObject A URL pointing to a player for a specific video. In general, this is the information in the src element of an embed tag and should not be the same as the content of the loc tag.
schema:featureList schema:SoftwareApplication Features or modules provided by this application (and possibly required by other applications).
schema:gameLocation schema:Game Real or fictional location of the game (or part of game).
schema:gamePlatform schema:VideoGame  or
schema:VideoGameSeries 		The electronic systems used to play video games.

schema:installUrl schema:SoftwareApplication URL at which the app may be installed, if different from the URL of the item.
schema:isBasedOnUrl schema:CreativeWork A resource that was used in the creation of this resource. This term can be repeated for multiple sources. For example, http://example.com/great-multiplication-intro.html.

schema:license schema:CreativeWork A license document that applies to this content, typically indicated by URL.
schema:logo schema:Organization  or

schema:paymentUrl schema:Order The URL for sending a payment.

schema:replyToUrl schema:UserComments The URL at which a reply may be posted to the specified UserComment.
schema:requirements schema:SoftwareApplication Component dependency requirements for application. This includes runtime environments and shared libraries that are not included in the application distribution package, but required to run the application (Examples: DirectX, Java or .NET runtime).
schema:roleName schema:Role A role played, performed or filled by a person or organization. For example, the team of creators for a comic book might fill the roles named 'inker', 'penciller', and 'letterer'; or an athlete in a SportsTeam might play in the position named 'Quarterback'. Supersedes schema:namedPosition.

schema:sport schema:SportsOrganization A type of sport (e.g. Baseball).
